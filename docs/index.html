Automation Toolkit for Justine Longla T.

A living collection of DevSecOps automation scripts used by Justine Longla T. — keeping blogs, docs, and project sites fast, safe, reproducible, and effortless to maintain.

Built with PowerShell & static HTML Designed for repeatable content workflows Safe to rerun, friendly for future contributors

How this toolkit fits in

These scripts live under .\tools\ and .\scripts\ and are wired into the blog/docs layout. They:

  • Create & find content (posts, projects, deep dives).
  • Validate front matter & tags so every page is well-formed.
  • Build & preview the static site before you ship.
  • Protect your history with safe rebase and “suture” utilities.

Core tools: how they work

A closer look at the scripts that keep this blog and docs repo moving: content creation, local previews, and safe Git history.

Content

tools\Add-Post.ps1

One command to create a new post skeleton, wired to the unified layout and card grid. 

pwsh .\tools\Add-Post.ps1
  • Prompts for title, slug, date, tags, and a short summary.
  • Creates posts\YEAR\MM\slug\index.html with the correct front matter.
  • Keeps the home page and card layout consistent by using the same HTML structure every time.

Why it’s reusable: any static site that stores posts by YEAR\MM\slug can drop in this script, adjust the template path, and get the same repeatable content workflow.

Build & preview

scripts\build-serve.ps1

Rebuilds the static site and starts a local preview server using production layout and cards. 

pwsh .\scripts\build-serve.ps1
  • Runs the full static build pipeline (metadata, index pages, sitemap, etc.).
  • Serves the generated site locally so you can click through pages before pushing.
  • Uses the same HTML output that ships to GitHub Pages / Vercel for realistic previews.

Why it’s reusable: any static HTML repo that produces a public or dist folder can plug in this pattern for a one-command “build and browse” experience.

Safety net

tools\Safe-Rebase.ps1

A guardrail around git rebase that keeps a recoverable backup of your branch. 

pwsh .\tools\Safe-Rebase.ps1 main
  • Verifies the working tree is clean before touching history.
  • Creates a backup branch (e.g. backup/main-before-rebase-YYYYMMDD).
  • Runs git fetch and git rebase against the target branch with clear logging.

Why it’s reusable: this script is repo-agnostic – you can drop it into any Git project where you want safer rebases and a documented rollback plan.

Safety

tools\Safe-Rebase.ps1

Guard-rail rebase: checks for a clean working tree, creates a backup branch, and only proceeds if everything is safe. Designed for human-friendly, low-stress history rewrites.

Git safety Backups
Safety

tools\git-suture.ps1

Stitches together mismatched changes after a risky operation (rebase, force push, manual file surgery). Helps you recover a known-good “reference state” without panic.

pwsh .\tools\git-suture.ps1
Recovery Point-in-time restore

Most scripts are idempotent: if they run into an unexpected state, they back out rather than leaving your repo half-changed. When in doubt, create a branch and let Safe-Rebase and git-suture watch your back.
